|---------|----------|----------|------|
| SCR001 | 游戏主场景 | main.tscn | 主要游戏画面 |

## 主游戏流程 (SCR001)

### 场景结构
```
Main (Node2D)
├── Player (CharacterBody2D) - 玩家飞机
├── EnemySpawnTimer (Timer) - 敌人生成计时器
├── SoundManager (Node) - 音效管理器
├── UI (CanvasLayer) - UI 层
│   ├── ScoreLabel (Label) - 分数显示
│   └── GameOverLabel (Label) - 游戏结束提示
└── (动态生成的敌人和子弹)
```

### 游戏循环流程

```
1. 游戏开始
   ↓
2. 玩家飞机出现在屏幕底部中央
   ↓
3. 敌人生成器开始工作 (每 1 秒生成一个敌人)
   ↓
4. 玩家控制飞机移动和射击
   ↓
5. 子弹击中敌人 → 敌人消失 + 分数 +100
   ↓
6. 敌人撞到玩家 → 玩家生命值 -1
   ↓
7. 生命值 = 0 → 游戏结束
   ↓
8. 显示游戏结束界面
   ↓
9. 按回车键 → 重新开始游戏
```

## 用户输入映射

| 输入 | 动作 | 游戏内效果 |
|------|------|------------|
| W / ↑ | move_up | 飞机向上移动 |
| S / ↓ | move_down | 飞机向下移动 |
| A / ← | move_left | 飞机向左移动 |
| D / → | move_right | 飞机向右移动 |
| 空格键 | shoot | 发射子弹 |
| 回车键 | ui_accept | 游戏结束后重新开始 |

## 详细流程说明

### 1. 游戏初始化流程

**触发**: 玩家按 F5 运行游戏 / 重新开始游戏

**步骤**:
1. 加载 main.tscn 场景
2. 初始化玩家飞机位置 (240, 600)
3. 初始化星空背景 (100 颗随机星星)
4. 启动敌人生成计时器 (间隔 1 秒)
5. 启动背景音乐
6. 显示分数 "分数：0"

**成功时**: 游戏开始，玩家可以控制飞机

**失败时**: 显示错误信息到控制台

---

### 2. 玩家移动流程

**触发**: 玩家按住 WASD 或方向键

**步骤**:
1. 检测输入方向
2. 计算归一化方向向量
3. 应用速度 (300 像素/秒)
4. 更新飞机位置
5. 限制位置在屏幕范围内 (32~448, 32~688)

**成功时**: 飞机流畅移动

**边界情况**:
- 到达屏幕边缘时停止
- 同时按多个方向键时斜向移动

---

### 3. 射击流程

**触发**: 玩家按空格键

**步骤**:
1. 检查射击计时器是否就绪
2. 如果就绪 → 实例化子弹场景
3. 设置子弹位置为玩家飞机前方
4. 将子弹添加到主场景
5. 播放射击音效
6. 启动射击计时器 (0.15 秒冷却)

**成功时**: 子弹向上飞行

**失败时**: 如果计时器未就绪，不发射

---

### 4. 敌人生成流程

**触发**: EnemySpawnTimer 超时 (每 1 秒)

**步骤**:
1. 生成随机 X 坐标 (50~430)
2. 实例化敌人场景
3. 设置敌人位置 (random_x, 80)
4. 连接 enemy_destroyed 信号
5. 将敌人添加到主场景
6. 播放敌人出现音效 (如果实现)

**成功时**: 敌人出现在屏幕上方

**边界情况**:
- 游戏结束时不生成敌人
- 确保生成位置不重叠

---

### 5. 敌人移动流程

**触发**: 每帧 (_process)

**步骤**:
1. 敌人水平移动 (速度 60 像素/秒)
2. 检测是否到达屏幕边缘
3. 如果到达边缘 → 反向移动
4. 如果超出屏幕底部 → 删除敌人

**成功时**: 敌人在屏幕上方左右移动

---

### 6. 碰撞检测流程

#### 6.1 子弹击中敌人

**触发**: 子弹的 Area2D 进入敌人的 Area2D

**步骤**:
1. 检测碰撞对象是否是敌人
2. 调用敌人的 take_damage(1)
3. 敌人生命值 -1
4. 如果生命值 <= 0:
   - 播放爆炸音效
   - 发射 enemy_destroyed 信号
   - 删除敌人
5. 删除子弹

**成功时**: 敌人消失，分数 +100

---

#### 6.2 敌人撞到玩家

**触发**: 玩家的 Area2D 进入敌人的 Area2D

**步骤**:
1. 检测碰撞对象是否是敌人
2. 玩家生命值 -1
3. 删除敌人
4. 播放爆炸音效
5. 检查玩家生命值是否 <= 0
6. 如果是 → 触发游戏结束

**成功时**: 玩家扣血，敌人消失

---

### 7. 游戏结束流程

**触发**: 玩家生命值 <= 0

**步骤**:
1. 设置 game_over = true
2. 停止敌人生成
3. 显示游戏结束标签
4. 显示最终分数
5. 等待玩家按回车键

**成功时**: 游戏结束界面显示

---

### 8. 重新开始流程

**触发**: 游戏结束状态下按回车键

**步骤**:
1. 检测输入 (ui_accept)
2. 重新加载当前场景
3. 所有变量重置
4. 分数重置为 0
5. 生命值重置为 3

**成功时**: 游戏重新开始

## UI 状态管理

### ScoreLabel (分数显示)

| 状态 | 显示内容 | 触发条件 |
|------|----------|----------|
| 正常 | "分数：X" | 游戏进行中 |
| 游戏结束 | "分数：X" (保持) | 游戏结束 |

### GameOverLabel (游戏结束提示)

| 状态 | 可见性 | 触发条件 |
|------|--------|----------|
| 隐藏 | false | 游戏进行中 |
| 显示 | true | 游戏结束 |

## 错误处理

### 常见错误场景

| 错误 | 处理方案 |
|------|----------|
| 音效加载失败 | 继续游戏，静音运行 |
| 场景加载失败 | 输出错误到控制台 |
| 碰撞检测失败 | 确保碰撞层设置正确 |
